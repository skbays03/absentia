# Why no LLM

> _Stub. To be written before any v1.0 push. Tracked in `DEFERRALS.md`._

The positioning, defended. In an AI-saturated tooling market, "deterministic, classical, explainable" is a real differentiator — but only if named explicitly. Without this doc, users assume there's a model hiding somewhere and trust collapses.

Outline:
- Determinism — same input, same output, every time
- Free explanations — every gap traces back to its rule, no post-hoc rationalization
- Latency — sub-100ms feedback for any TUI interaction
- Cost — runs locally, no API bills
- Trust — pattern miners produce the rule as a byproduct of finding the gap; LLMs produce justifications that can hallucinate
- Differentiation — most "AI code review" tools are crowded; this one isn't
- Where embeddings/LLMs *might* enter (cross-document semantic matching, NL query shell) — clearly bounded as optional, not core
